Étape 1 : Analyse

Description : L’étape d’analyse vise à comprendre les besoins du client, les contraintes du projet et à définir clairement les périmètres fonctionnels et techniques. Elle permet d’éviter les dérives en cours de projet.

Outils :

  • JIRA – Gestion des exigences
  • Diagrammes UML – Cas d'utilisation
  • Entretiens, SWOT, matrices d'analyse
Étape 2 : Planification

Description : Cette phase consiste à organiser le travail à faire : ordonnancer les tâches, évaluer les charges, les délais, les ressources et construire un planning fiable.

Outils :

  • Microsoft Project – Diagrammes de Gantt
  • WBS – Work Breakdown Structure
  • Primavera – Planification industrielle multi-projets
Étape 3 : Exécution

Description : C’est la phase d’implémentation du projet où les solutions sont développées, les équipes sont coordonnées, et le planning est suivi en temps réel.

Outils :

  • Kanban / Scrum Board (via JIRA)
  • Daily meetings (Agile)
  • RACI Matrix – Répartition des rôles
Étape 4 : FAT

Description : Test des solutions en usine avant livraison. Elle valide le bon fonctionnement du système dans un environnement contrôlé, selon les exigences définies.

Outils :

  • Check-list fonctionnelle
  • JIRA pour le suivi des anomalies
  • TestLink – outil de validation
Étape 5 : SAT

Description : Phase de test et validation sur site client. Elle permet de s'assurer que les solutions sont déployées correctement et conformes aux attentes.

Outils :

  • Plan de déploiement
  • Ansible, Puppet – automatisation
  • Formulaire de réception client
Étape 6 : Clôture

Description : La phase de clôture consiste à livrer officiellement le projet, former les utilisateurs, archiver la documentation et tirer des leçons pour les projets futurs.

Outils :

  • Rapport de clôture
  • Microsoft Project (archivage)
  • Réunion REX (Retour d'expérience)
